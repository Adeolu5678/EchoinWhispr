# Part III: Data Models & Database Schema

## 3.1. Database Schema (Convex)

The application uses Convex as its backend and database. The schema is defined in `convex/schema.ts`.

### Users Table (`users`)

| Field | Type | Description |
|-------|------|-------------|
| `clerkId` | `string` | Unique identifier from Clerk (indexed). |
| `role` | `string` | "entrepreneur" or "investor". |
| `firstName` | `string` | First Name (Private). |
| `lastName` | `string` | Last Name (Private). |
| `email` | `string` | Email address (Private). |
| `professionalBio` | `string` | Short professional biography (Public). |
| `linkedinUrl` | `string` | Link to LinkedIn profile (Private until connected). |
| `avatarUrl` | `string` | Profile picture URL. |
| `isVerified` | `boolean` | Identity verification status. |

### Certifications Table (`certifications`)

| Field | Type | Description |
|-------|------|-------------|
| `userId` | `Id<"users">` | Owner of the certification. |
| `title` | `string` | Name of the certification (e.g., "AWS Solutions Architect"). |
| `imageUrl` | `string` | URL of the certificate image. |
| `status` | `string` | "pending", "verified", "rejected". |
| `verifiedAt` | `number` | Timestamp of verification. |

### Projects Table (`projects`)

| Field | Type | Description |
|-------|------|-------------|
| `ownerId` | `Id<"users">` | ID of the entrepreneur who owns the project. |
| `workspaceId` | `Id<"workspaces">` | Link to the associated workspace. |
| `title` | `string` | Project name. |
| `tagline` | `string` | Short elevator pitch. |
| `description` | `string` | Full project description. |
| `industry` | `string` | e.g., "Fintech", "Health", "AI". |
| `fundingGoal` | `number` | Amount seeking in USD. |
| `equityOffered` | `number` | Percentage of equity offered. |
| `status` | `string` | "draft", "published", "funded", "closed". |
| `logoUrl` | `string` | Project logo URL. |
| `pitchDeckUrl` | `string` | URL to PDF pitch deck (optional). |
| `createdAt` | `number` | Timestamp. |

### Workspaces Table (`workspaces`)

| Field | Type | Description |
|-------|------|-------------|
| `projectId` | `Id<"projects">` | The project this workspace belongs to. |
| `name` | `string` | Workspace name (same as Project Title). |
| `members` | `array<Id<"users">>` | List of users with access. |
| `createdAt` | `number` | Timestamp. |

### Applications Table (`applications`)

| Field | Type | Description |
|-------|------|-------------|
| `applicantId` | `Id<"users">` | ID of the user applying. |
| `projectId` | `Id<"projects">` | ID of the target project. |
| `role` | `string` | e.g., "Co-founder", "CTO", "Investor". |
| `message` | `string` | Cover letter / intro message. |
| `status` | `string` | "pending", "interviewing", "accepted", "rejected". |
| `createdAt` | `number` | Timestamp. |

### Conversations Table (`conversations`)

| Field | Type | Description |
|-------|------|-------------|
| `participantIds` | `array<Id<"users">>` | IDs of users in the conversation. |
| `workspaceId` | `Id<"workspaces">` | (Optional) If this is a workspace chat. |
| `applicationId` | `Id<"applications">` | (Optional) If this is an interview chat. |
| `type` | `string` | "direct", "workspace_general", "interview". |
| `lastMessageId` | `Id<"messages">` | Pointer to the last message for sorting. |
| `updatedAt` | `number` | Timestamp of last activity. |

### Messages Table (`messages`)

| Field | Type | Description |
|-------|------|-------------|
| `conversationId` | `Id<"conversations">` | Parent conversation ID (indexed). |
| `senderId` | `Id<"users">` | ID of the sender. |
| `content` | `string` | Text content of the message. |
| `imageUrl` | `string` | (Optional) Attached image. |
| `createdAt` | `number` | Timestamp. |

### Milestones Table (`milestones`)

| Field | Type | Description |
|-------|------|-------------|
| `projectId` | `Id<"projects">` | The project this milestone belongs to. |
| `title` | `string` | e.g., "MVP Launched". |
| `description` | `string` | Details of the achievement. |
| `date` | `number` | Date of achievement. |
| `verified` | `boolean` | Whether the system/admin verified this. |

### Bounties Table (`bounties`)

| Field | Type | Description |
|-------|------|-------------|
| `projectId` | `Id<"projects">` | The project offering the bounty. |
| `title` | `string` | Task name. |
| `description` | `string` | Task details. |
| `reward` | `string` | e.g., "$500", "0.1% Equity". |
| `status` | `string` | "open", "claimed", "completed", "paid". |
| `assigneeId` | `Id<"users">` | (Optional) User working on it. |

### Soft Circles Table (`soft_circles`)

| Field | Type | Description |
|-------|------|-------------|
| `projectId` | `Id<"projects">` | Target project. |
| `investorId` | `Id<"users">` | Investor showing interest. |
| `amount` | `number` | Indicated investment amount. |
| `status` | `string` | "interested", "committed", "withdrawn". |

### Vouches Table (`vouches`)

| Field | Type | Description |
|-------|------|-------------|
| `voucherId` | `Id<"users">` | User giving the vouch. |
| `targetId` | `Id<"users">` | User receiving the vouch. |
| `relationship` | `string` | e.g., "Former Co-founder", "Investor". |
| `text` | `string` | Endorsement message. |

## 3.2. Backend Functions (Convex)

### User Functions

- **`register`**: Creates a new user record with a specific role.
- **`updateProfile`**: Updates bio, LinkedIn, etc.
- **`uploadCertification`**: Submits a cert for review.
- **`verifyCertification`**: (Admin) Approves a cert.

### Project & Workspace Functions

- **`createProject`**: Creates a project AND a workspace.
- **`joinWorkspace`**: Adds a user to the workspace members list.
- **`getWorkspaceDetails`**: Fetches members and chat history.

### Application Functions

- **`applyToProject`**: Submits an application.
- **`startInterview`**: Creates a conversation linked to the application.
- **`acceptApplicant`**: Updates status and adds user to workspace.

### Messaging Functions

- **`sendMessage`**: Creates a new message (supports images).
- **`getMessages`**: Paginated query for messages.

## 3.3. File Storage

- **Project Assets**: Logos and Pitch Decks are stored in Convex File Storage.
- **User Avatars**: Profile pictures.
