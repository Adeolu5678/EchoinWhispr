# Part V: Non-Functional Requirements (NFRs)

## 5.1. Security & Privacy

- **Data Protection:** All user data in transit must be encrypted via TLS.
- **Authentication:** All private routes must be protected via Clerk middleware.
- **Authorization:** Convex Row Level Security (RLS) must strictly enforce that only Project Owners can view applications and only Investors can initiate contact.
- **Identity & Trust:**
  - **Private Identity:** Real names, emails, and phone numbers are hidden by default.
  - **Public Persona:** Profiles display "Professional Bio", "Skills", and **"Verified Certifications"**.
  - **Verification:** The "Blue Tick" is the primary signal of trust, not the user's real name.

## 5.2. Performance & Scalability

- **Real-time Latency:** Message delivery and application notifications should appear instantaneous (< 100ms) using Convex's WebSocket-based subscription model.
- **Database Indexing:** Queries for "Project Feed" must be optimized with compound indexes (Industry + Stage) to handle high volume.
- **Asset Optimization:** Pitch decks (PDFs) and Logos must be optimized for fast retrieval.

## 5.3. Usability & Accessibility (UX)

- **Professional Aesthetic:** The interface should be clean, professional, and "high-trust". Avoid playful or "gamified" elements suitable for social apps.
- **Feedback:** Immediate visual feedback for all actions (publishing projects, sending applications).
- **Accessibility:** The app should follow WCAG 2.1 guidelines.

## 5.4. Reliability

- **Uptime:** Rely on Convex and Vercel's high availability infrastructure (99.9% SLA).
- **Data Integrity:** Financial data (Funding Goals, Equity) must be handled with precision.
