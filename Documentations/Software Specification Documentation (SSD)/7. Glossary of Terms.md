# Part VII: Glossary of Terms

- **CID (Content Identifier):** A hash address for a file on IPFS. Used for both the encrypted message object and the raw image file.
- **Conversation ID (convo_id):** A client-side-only UUID used to group messages from the same HCS thread into a single chat window.
- **ECIES (Elliptic Curve Integrated Encryption Scheme):** The encryption standard used for securing messages, allowing encryption with a public key and decryption only with the corresponding private key.
- **HCS (Hedera Consensus Service):** Hedera's service for high-throughput, low-cost, verifiable event logging. Used here as a message broker.
- **Mail Hash (mailHash):** A user's permanent, public, static, and anonymous "mailbox" hash. Stored on the smart contract. Used only to receive the first message (invitation) of a new conversation.
- **Message Object (Plaintext):** A JSON-structured object containing the text and/or image CID of a message. This entire object is encrypted before being stored on IPFS.
- **Mirror Node:** A node on the Hedera network that stores and provides access to all historical transaction data.
- **Persona:** The user's anonymous, on-chain profile (Career, Interests, Mood).
- **Reply Hash (reply_hash):** A temporary, random, and single-use hash generated by a sender. It is included in an HCS message and serves as the to_hash for the next reply in that specific conversation. This is the core mechanism for two-way anonymous routing.
- **Whisper:** The first message sent to a user's mailHash that initiates a new, anonymous conversation.
