# Part I: Foundational Project Specification

## 1.1. Introduction and Document Purpose

This document is the definitive Software Specification Document (SSD) for the "Launchpad" application. It serves as the single source of truth for all stakeholders.
  - **Pain Point:** Doesn't know how to structure a pitch or where to find serious investors.

- **Persona 2: The "Angel" (Investor)**
  - **Description:** An accredited investor or VC scout looking for early-stage opportunities.
  - **Goal:** To efficiently source and vet high-potential startups.
  - **Pain Point:** Overwhelmed by low-quality inbound requests and lack of standardized data.

- **Persona 3: The "Builder" (Talent)**
  - **Description:** A developer, designer, or marketer looking to join an early-stage team for equity.
  - **Goal:** To find a project that matches their skills and passion.

## 1.4. User Journeys (Epics & Use Cases)

### Use Case 1: Project & Workspace Creation
- **Actor:** Entrepreneur.
- **Trigger:** User clicks "Start New Project."
- **Steps:**
  1. User enters Project Essentials (Name, Tagline, Logo).
  2. User defines The Pitch (Problem, Solution, Market).
  3. User sets The Ask (Funding Goal, Equity).
  4. User publishes the project.
  5. **System automatically creates a Workspace.**
- **Post-condition:** Project is live, and a collaborative workspace is ready.

### Use Case 2: Certification Verification
- **Actor:** Entrepreneur / Talent.
- **Trigger:** User uploads a certification image.
- **Steps:**
  1. User submits cert details.
  2. Admin reviews the submission.
  3. Admin approves.
- **Post-condition:** A "Verified" blue tick appears next to the cert on the user's profile.

### Use Case 3: Deal Discovery & Filtering
- **Actor:** Investor.
- **Trigger:** User visits the "Deal Flow" dashboard.
- **Steps:**
  1. User filters by Industry, Stage, or Funding Goal.
  2. System displays a list of matching projects.
  3. User clicks a project to view the full pitch.
- **Post-condition:** Investor has detailed project information.

### Use Case 4: Connection & Negotiation
- **Actor:** Investor.
- **Trigger:** User clicks "Contact Founder" on a project.
- **Steps:**
  1. A private conversation channel is created.
  2. Investor and Founder discuss terms.
- **Post-condition:** Direct line of communication established.

### Use Case 5: Team Application & Interview
- **Actor:** Talent (Builder).
- **Trigger:** User clicks "Apply to Join" on a project.
- **Steps:**
  1. User submits profile and note.
  2. Founder receives application.
  3. Founder starts an "Interview Chat" to verify skills.
  4. Founder accepts application.
- **Post-condition:** User is added to the project workspace.

## 1.5. Project Scope

### In Scope (MVP Features):
- **MVP 1:** Dual-Role Authentication (Entrepreneur / Investor).
- **MVP 2:** Project Creation Wizard ("The Forge") + **Workspace Auto-creation**.
- **MVP 3:** Project Marketplace & Advanced Filtering.
- **MVP 4:** Team Application System + **Interview Chats**.
- **MVP 5:** Direct Messaging (Investor-Founder, Team Chat, Workspace Chat).
- **MVP 6:** User Profiles (**Verified Certifications**, Private Identity).

### Advanced Features (Phase 2 - "The Ecosystem"):
- **Living Pitch Decks:** Founders post "Milestone Updates" (e.g., "MVP Launched") to show traction. Investors "Follow" projects to get updates.
- **Micro-Bounties:** Founders post small tasks (e.g., "Design Logo") to test talent before hiring.
- **AI Matching:** Proactive suggestions for Investor-Project and Founder-Talent matches based on skills and thesis.
- **Soft Circle Commitments:** Investors indicate non-binding interest amounts to build momentum (FOMO).
- **Integrated Legal Templates:** Standardized, auto-filled agreements (NDAs, SAFEs) generated within Workspaces.
- **Vouched Reputation:** Verified users can "vouch" for others, digitizing the "warm intro" and building a trust graph.

### Out of Scope (Future Iterations):
- In-app Payment Processing for Investments (Regulatory complexity).
- DAO governance structures.
