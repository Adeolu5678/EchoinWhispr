# Part IV: Detailed Functional Workflows

## 4.1. Workflow: Project & Workspace Creation (The Forge)

This workflow describes how an Entrepreneur transforms an idea into a published project and active workspace.

### Steps:

1. **Initiation:** User clicks "Create Project" from the dashboard.
2. **Step 1: Essentials:**
   - User inputs Project Name, Tagline, and Industry.
   - User uploads a high-res logo.
3. **Step 2: The Pitch:**
   - User fills in "Problem", "Solution", and "Market".
   - (Optional) User uploads a PDF Pitch Deck.
4. **Step 3: The Ask:**
   - User sets Funding Goal ($) and Equity Offered (%).
5. **Review & Publish:**
   - User clicks "Publish".
   - Client calls `mutation projects.create(data)`.
   - **System Action:**
     - Creates `projects` record.
     - Creates `workspaces` record (Name = Project Name).
     - Adds Owner to `workspaces.members`.
     - Sets status to `published`.

## 4.2. Workflow: Applying & Interviewing

This workflow describes the hiring/teaming process.

### Steps:

1. **Application:** User applies to a project (Role: "Developer").
2. **Notification:** Founder receives an alert.
3. **Interview:**
   - Founder views application.
   - Founder clicks "Start Interview".
   - Client calls `mutation conversations.create(applicationId)`.
   - A private chat opens with the Application as the header context.
4. **Verification:** Founder chats with applicant to verify skills.
5. **Acceptance:**
   - Founder clicks "Accept Applicant".
   - System updates application status to `accepted`.
   - System adds applicant to `workspaces.members`.
   - Applicant gains access to the Workspace.

## 4.3. Workflow: Workspace Communication

### General Chat
- Accessible to all members of the workspace.
- History is persistent (new members see past messages).

### Direct Messaging
- Members can click on any other member in the workspace to start a private DM.

## 4.4. Workflow: Certification Verification

### Steps:

1. **Upload:** User uploads a certificate image and title via Profile.
2. **Pending:** Status is set to `pending`.
3. **Review:** Admin views the "Verification Queue".
4. **Decision:**
   - Admin clicks "Verify" -> Status `verified` -> Blue Tick appears on user profile.
   - Admin clicks "Reject" -> Status `rejected` -> User notified.

## 4.5. Workflow: Deal Negotiation (Direct Messaging)

This workflow describes the private communication channel between an Investor and a Founder.

### Steps:

1. **Initiation:** Investor clicks "Contact Founder" on a project page.
2. **Check Existing:** System checks for an existing conversation between these two users for this specific project.
3. **Create Channel:**
   - If none exists, client calls `mutation conversations.create(founderId, projectId)`.
   - System creates a new conversation of type `direct`.
4. **Messaging:**
   - Standard real-time messaging ensues (see `messages` table).
   - Both parties can share documents/images via Convex Storage.

## 4.6. Workflow: Search and Discovery

- **Trigger:** Investor selects filters (e.g., "Fintech", "Seed Stage").
- **Query:** Client calls `query projects.list(filters)`.
- **Display:** Results are displayed in a grid/list view.
- **Real-time:** If a new project is published that matches the filter, it appears instantly.

## 4.7. Workflow: Milestone Updates (Living Pitch Deck)

### Steps:
1. **Post:** Founder posts a milestone (e.g., "First 100 Users") with proof (image/link).
2. **Notify:** System notifies all Investors who "Follow" the project.
3. **Display:** Milestone appears on the Project Detail page timeline.
4. **Impact:** Increases project "Traction Score" (internal metric).

## 4.8. Workflow: Micro-Bounties

### Steps:
1. **Create:** Founder posts a bounty (e.g., "Fix CSS Bug", Reward: $50).
2. **Claim:** Talent clicks "Claim Bounty".
3. **Submit:** Talent submits work (PR link or file).
4. **Review:** Founder reviews and approves.
5. **Reward:** System marks bounty as `paid` and updates Talent's profile with "Bounty Completed".

## 4.9. Workflow: Soft Circle Commitment

### Steps:
1. **Interest:** Investor clicks "Soft Circle" on a project.
2. **Input:** Investor enters amount (e.g., $25k).
3. **Display:** Project's "Committed" progress bar updates (e.g., "60% Soft Circled").
4. **Privacy:** Individual investor names are hidden from public, visible only to Founder.

## 4.10. Workflow: Vouching

### Steps:
1. **Initiate:** User A visits User B's profile.
2. **Vouch:** User A clicks "Vouch for [Name]".
3. **Context:** User A selects relationship (e.g., "Worked together") and writes a short note.
4. **Display:** Vouch appears on User B's profile.

## 4.11. Workflow: App Settings & Profile Management

### Steps:
1. **Access:** User clicks "Settings" from the navigation menu.
2. **Profile Settings:**
   - User updates Display Name, Bio, and Avatar.
   - User manages "Skills" and "Interests" tags.
3. **Account Settings:**
   - User manages Notification preferences (Email, Push).
   - User toggles "Dark/Light Mode" (Theme).
4. **Privacy:**
   - User controls visibility of their profile (Public/Private).
