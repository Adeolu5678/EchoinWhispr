These foundational assumptions, based on our discussions, govern all subsequent design decisions.

### 1. Professional & Premium
The platform is a high-stakes environment for business. The UI must reflect trust, clarity, and sophistication.
- **Glassmorphism:** Used to create depth and a modern, premium feel.
- **Verification:** "Blue Ticks" and "Vouched" badges are central to building trust.

### 2. Role-Based Experience
The UX bifurcates early into two distinct paths:
- **Entrepreneurs:** Focus on "Project Creation," "Updates," and "Team Building."
- **Investors:** Focus on "Deal Flow," "Due Diligence," and "Portfolio Management."

### 3. The "Living Pitch Deck"
Projects are not static. The UX prioritizes **Milestone Updates** and **Real-time Progress** over static text.
- **Timeline UI:** A central component for every project.
- **Notifications:** Investors are notified of progress, not just "news."

### 4. Frictionless Onboarding
- **Web2 Auth:** Email/Social login (Clerk) is the primary entry point.
- **Progressive Profiling:** Users can sign up quickly and build their detailed profile (and get verified) later.
"Whisper Request" Inbox: To manage new conversations and prevent spam, the inbox is bifurcated.
"Requests" Tab: All new, incoming Whispers land here. A user must "Accept" a request.
"Chats" Tab: All "Accepted" conversations and ongoing replies live here.
MVP Scope: "Block" functionality is out of scope for MVP. A user can ignore a request by simply not accepting it.
Managing Decentralization (Syncing): We will use explicit loading states to manage user expectations during decentralized data synchronization (per HCS and client-side search).
Inbox Sync: "Please, wait while your messages are being updated..."
Search Sync: "Syncing user directory..."
